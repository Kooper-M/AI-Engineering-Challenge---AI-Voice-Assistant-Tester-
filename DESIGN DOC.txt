Design Doc
The program uses two main APIs: Twilio for outbound calling, call recording, and Conversation Relay, and OpenAI for the LLM that generates patient responses and analyzes completed transcripts. The make_call.py file starts an outbound call using the Twilio API and gives Twilio the public ngrok URL for the /twiml route. Ngrok exposes the local Uvicorn/FastAPI server to the internet so Twilio can reach it.

The /twiml route returns a ConversationRelay response, which tells Twilio to connect the call to the /ws WebSocket route. Once connected, Twilio sends transcribed speech from the phone call to the WebSocket. The app appends each transcribed agent message to the current call session history and sends that history to the OpenAI API. OpenAI generates a patient-style response based on the system prompt, scenario prompt, and conversation history. The app sends that response back to Twilio over the WebSocket so it can be spoken into the call. Both the agent’s transcribed speech and the patient bot’s text responses are saved to a transcript file with timestamps.

After the call ends, Twilio processes the call recording and sends a POST request to the /recording-complete route. This callback includes the recording URL, which the app uses to download the MP3 from Twilio and save it locally. After the transcript is saved, the app also sends the transcript to OpenAI for analysis. The analysis looks for bugs, failed scenario goals, turn-taking issues, transcription problems, and unnatural responses. The final analysis is saved to a text file, using the same call SID to connect the transcript, recording, and analysis files.
